## Обо мне

Я увлечён тестированием как инструментом обеспечения надёжности ПО. Мои ключевые принципы:

- **Тест-дизайн - основа всего:** перед автоматизацией - чёткие сценарии.
- **Безопасность - не опция:** никогда не коммичу секреты, использую `.env` и GitHub Secrets.
- **Данные должны совпадать:** если API говорит одно, а БД другое - это баг.
- **Автоматизация - помощник, а не цель:** пишу скрипты с поддержкой AI-ассистента, которые реально экономят время.

Помимо тестирования коммерческих продуктов, я развиваю собственную платформу для QA-инженеров - **NeiroQA**, объединяющую в настоящее время инструменты нагрузочного, security-тестирования (в разработке) и управления знаниями команды под единым SSO. Это рабочий способ изучать новые технологии на практике.

В своей работе использую AI-инструменты (Cursor, Copilot) для ускорения рутинных задач: генерация тестовых данных, написание boilerplate-кода, первичный анализ логов. При этом всю архитектуру решений, тест-дизайн и финальную валидацию результатов выполняю самостоятельно.

Сейчас углубляюсь в Playwright, API-тестирование, валидацию данных, а также изучаю практики CI/CD и интеграции с системами отслеживания задач.

## Собственные продукты: AI-платформа для QA (NeiroQA)

В свободное время я развиваю собственную платформу для QA-инженеров и DevSecOps - **[neiroqa.ru](https://neiroqa.ru)**. Это self-hosted экосистема для нагрузочного и security-тестирования, а также управления знаниями команды под единым SSO.

**Архитектура и стек:** Docker, PostgreSQL, SSO, AI-интеграция (LLM для анализа метрик и уязвимостей).

### 🔹 PerfInsight AI (Нагрузочное тестирование)

- **Суть:** Дашборд и UI для запуска тестов **k6**.
- **Фишки:** Сравнение прогонов (baseline / current), SSE-логи, HTML-отчёты и **AI-вердикт** с присвоением приоритетов P0-P3 на основе метрик.
- 🔗 [Попробовать PerfInsight](https://piai.neiroqa.ru)

### 🔹 SecInsight AI (Security-тестирование) _(В разработке)_

- **Суть:** Агрегация отчётов из **OWASP ZAP** и **Burp Suite**.
- **Фишки:** Дашборд severity/CWE, delta-сравнение между релизами (NEW/FIXED/REGRESSED) и AI-анализ уязвимостей.
- 🔗 [Подробнее о SecInsight](https://neiroqa.ru)

### 🔹 LinkManager (База знаний QA)

- **Суть:** Централизованное управление закладками команды с тегами и AI-поиском.
- **Фишки:** Web и Desktop клиенты, импорт из браузера, проверка битых ссылок.
- 🔗 [Открыть LinkManager](https://lm.neiroqa.ru)

**Что реализовано:**

- Спроектировал и развернул на VPS экосистему из 3-х сервисов с использованием Docker и PostgreSQL
- Настроил единый механизм SSO и ролевую модель
- Интегрировал AI (LLM) для автоматического анализа результатов нагрузочного тестирования и security-сканов с автоматическим присвоением приоритетов P0-P3
- Реализовал функционал Compare (baseline vs current) для отслеживания деградации производительности и появления новых уязвимостей между релизами
- Разработал Desktop и Web-клиенты для LinkManager с AI-поиском и валидацией URL

### Мобильное тестирование. Отчет о тестировании:

    1. При тестировании мобильного приложения «Лемана ПРО» работа велась на физическом устройстве
       (Xiaomi 12) - это критически важно для проверки реального пользовательского опыта.
    2. Я проверил 5 модулей, включая тестирование безопасности. И здесь - ключевая находка:
       приложение оказалось уязвимо к SQL-инъекциям, что приводило к сбою интерфейса. Это критическая
       уязвимость. Мой вывод как тестировщика: билд не может быть рекомендован к выпуску до устранения
       этой проблемы. Найдено 3 дефекта, 2 из них - критические.
    3. Дополнительно я провел тестирование на эмуляторе и закрепил тезис, что тестирование на физических
       устройствах остается незаменимым и критически важным этапом обеспечения качества ПО, особенно для
       мобильных. Мой подход показал, что есть категории дефектов, которые невозможно или крайне сложно
       обнаружить виртуальными средствами.
